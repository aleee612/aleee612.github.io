#### 定长和变长指令
**定长指令集**：所有指令的长度完全相同  
    → 典型架构：ARM
    → 优点：解码简单、高效  
    → 缺点：指令中可能出现浪费空间的无效字段
- 对于地址字段短于定长的指令，则使用无效字段填充
- 若长于，则使用 多字指令 或 简介寻址（使用内存字段替代真正操作数地址）
**变长指令集**：不同指令具有不同长度
    → 典型架构：x86
    → 优点：编码更紧凑，能支持更复杂操作  
    → 缺点：解码过程复杂，需要准确识别指令边界
- 识别指令边界：使用指令前缀
##### 操作码扩展编码
在定长指令中，地址越长，则操作码越短
一般认为：16位指令字中，
	三地址指令 4 位操作码  
	二地址指令 8 位操作码  
	一地址指令 12 位操作码

### 寻址方式
| 寻址方式        | 含义                      | 操作数位置 | 优点             | 缺点/适用场景        | 计算有效地址 (EA) 示例                                   |
| ----------- | ----------------------- | ----- | -------------- | -------------- | ------------------------------------------------ |
| **立即寻址**    | 指令中直接给出操作数              | 指令本身  | 取数快，简单         | 不能修改操作数，只能用于常数 | `EA` 不需要，操作数 = 指令中值                              |
| **直接寻址**    | 地址字段给出操作数的内存地址          | 内存    | 简单，直观          | 地址字段长，需要更多位    | `EA = 地址字段`                                      |
| **间接寻址**    | 地址字段指向一个存储真正地址的内存单元     | 内存    | 节省地址位，可访问大地址空间 | 访问慢，多次内存访问     | `EA = M[地址字段]`                                   |
| **寄存器寻址**   | 操作数在寄存器中                | 寄存器   | 访问快            | 寄存器数量有限        | `EA = Rn`（寄存器编号）                                 |
| **寄存器间接寻址** | 地址在寄存器里，寄存器存放有效地址       | 寄存器   | 快，适合遍历数组       | 寄存器有限          | `EA = [Rn]`（寄存器内容作为地址）                           |
| **相对寻址**    | 地址 = PC + 指令中偏移         | 内存    | 支持分支，程序可移动     | 偏移范围受限         | `EA = PC + offset`                               |
| **基址寻址**    | 地址 = 基址寄存器 + 偏移         | 内存    | 支持大内存区，灵活      | 需占用寄存器         | `EA = BR + offset`                               |
| **变址寻址**    | 地址 = 变址寄存器 + 偏移（常用于数组）  | 内存    | 支持数组访问         | 寄存器有限          | `EA = XR + offset`<br>`操作数 = M[EA]`*变址寻址只访问一次内存* |
| **基址+变址寻址** | 地址 = 基址寄存器 + 变址寄存器 + 偏移 | 内存    | 支持复杂数据结构       | 复杂，寄存器占用多      | `EA = BR + XR + offset`                          |
- 变址寄存器：专门计算具体偏移用（算上元素大小）
- 基址寄存器：存放数据结构起始位置
##### 访问内存次数
| 寻址方式     | 内存访问次数  |
| -------- | ------- |
| 直接寻址     | 1 次     |
| 变址寻址     | 1 次     |
| 寄存器间接    | 1 次     |
| **间接寻址** | **2 次** |

### 错题
> 下列选项中，属于指令集体系结构（ISA）规定的内容是（ ）。
	Ⅰ. 指令字格式和指令类型
	Ⅱ. CPU 的时钟周期
	Ⅲ. 通用寄存器个数和位数
	Ⅳ. 加法器的进位方式
	A.仅Ⅰ、Ⅱ       B.仅Ⅰ、Ⅲ       C.仅Ⅱ、Ⅳ       D.仅Ⅰ、Ⅲ、Ⅳ
- 加法器的进位方式涉及电路设计，不由指令集规定
- 通用寄存器认为是与机器指令有关，因此由ISA规定
> 某机器字长 16 位，主存按字节编址，转移指令采用相对寻址，由两个字节组成，第一字节为操作码字段，第二字节为相对位移量字段。假定取指令时，每取一个字节 PC 自动加 1。若某转移指令所在主存地址为 2000H，相对位移量字段的内容为 06H，则该转移指令成功转移后的目标地址是（）。
	A.2006H        B.2007H        C.2008H        D.2009H
- EA = (PC)+A
- 每取一个字节认为pc++，由于主存按字编址，则取完转移指令时pc为2002H
- 由于转移指令使用的相对寻址，则地址 = PC + 指令中偏移（06H）
- 由上述可得，最终目标地址为2008H
