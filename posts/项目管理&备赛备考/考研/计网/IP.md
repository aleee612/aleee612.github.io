### 子网划分
> 从主机部分 “借用” 若干位，加入到网络部分，用来表示新的子网
![[Pasted image 20260312111359.png]]
#### VLSM 变长子网划分
> 实质上是一种 前缀编码 方式，要求各子网的前缀互不包含，以确保地址空间的唯一划分
![[Pasted image 20260312111318.png]]
#### FLSM 固定长度子网划分
- 每个子网使用相同的子网掩码
- 每个子网拥有相同数量的 IP 地址
> 每个子网都对应于二叉树的 **最底层叶子结点**
![[Pasted image 20260312111619.png]]
### 子网聚合
**聚合条件**：
- **地址对齐**：聚合后的网络地址必须是其新掩码下的网络起始地址（低位必须为 0）；
- **掩码缩短**：聚合的结果一定是更短的前缀，比如从 /24 聚合成 /22。
> 将若干个叶子结点合并为一个中间结点

### IP 组播
> 允许一台发送器将数据报发送到多个接收器，而不是传统的单播（点对点通信）模式
![[Pasted image 20260312114551.png]]

| 概念                          | 含义                                          | 关键特点                                                             |
| --------------------------- | ------------------------------------------- | ---------------------------------------------------------------- |
| **组播组（Multicast Group）**    | IP 组播通信由一个或多个组播组组成，每个组有唯一的组播地址              | IPv4 组播地址范围为 **224.0.0.0 – 239.255.255.255（D 类地址）**，多个接收器可以订阅同一组 |
| **发送器（Sender）**             | 向组播组发送数据报的设备                                | 发送器只发送 **一次数据**，网络设备负责复制并分发                                      |
| **接收器（Receiver）**           | 订阅某个组播组并接收数据的设备                             | 可以是 **单个主机、多个主机或路由器**                                            |
| **IGMP 协议**                 | Internet Group Management Protocol，用于管理组播成员 | 主机通过 IGMP **加入或离开组播组**，路由器据此确定数据发送对象                             |
| **组播路由（Multicast Routing）** | 负责将组播数据从发送器传送到接收器                           | 路由器根据 **IGMP 信息和组播路由表**，只向需要的网络分支转发                              |
| **组播范围（Multicast Scope）**   | 组播地址根据用途划分不同范围                              | 常见类型：**永久组播地址、临时组播地址、本地链路组播地址**                                  |
$$组播 = 组地址 + IGMP管理 + 组播路由转发$$
