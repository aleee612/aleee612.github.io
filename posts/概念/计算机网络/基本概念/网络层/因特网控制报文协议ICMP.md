### 用途
- 差错报告，如ping使用的就为ICMP报文
- ICMP源抑制报文 -- 拥塞控制
### 传输方式
- 类似[[开放最短路优先OSPF|OSPF]]，是直接封装在IP数据报中的（IP的数据部分）而不需要别的层的首部
### 结构
- 一个类型字段和一个编码字段，包含引起该ICMP报文首次生成的IP数据报的首部和前8个字节
#### 首部：
| 结构           | 长度  | 作用               |
| ------------ | --- | ---------------- |
| type 类型      | 8位  | 指定类型（差错 / 查询）    |
| code 代码（编码）  | 8位  | 细分特定类型 （具体差错原因等） |
| Checksum 校验和 | 16位 | 验证 ICMP 是否损坏     |
| Content 其他字段 | 不等  | 取决于上述三结构         |

### 类型
![[Pasted image 20251024215541.png]]
> 主要分为 差错和查询两种
- **差错报文**：用于 **报告** 网络通信过程中出现的各种 **错误** 。
- **查询报文**：用于 **诊断** 或网络信息查询，主要用于网络测试和管理 。
### 应用
#### ping
作用：测量往返时延RTT，并检测是否有 数据包丢失
报文类型：查询报文
#### traceroute
作用：跟踪数据包从源到目标的 **路径**，显示沿途经过的路由器（**跳点**）及其 **延迟**
报文类型：差错报文
1. **逐步增加 TTL**：发送 UDP 数据包（或 ICMP Echo Request），从 TTL=1 开始，每次递增 1。每台路由器在转发数据包时将 **TTL** 减 1。当 **TTL** 减为 0 时，路由器丢弃数据包并返回一个 **ICMP Time Exceeded** 消息（类型 _11_，代码 _0_）
2. **记录跳点**：记录发送 **Time Exceeded** 消息的路由器 IP 地址和响应时间
3. **显示路径**：将每跳的路由器 IP 和 **延迟** 显示出来，构成从源到目标的完整 **路径**
